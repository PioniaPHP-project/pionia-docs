DeskFlow API tutorial

Who this is for#

You want one continuous tutorial — like Django’s first steps — that starts at an empty folder and ends with real rows in a database. No jumping between disconnected guides until you have something working.

By the end of this page you will have DeskFlow: Northwind Studio’s task board API, running on your machine, with task.list reading from SQLite and task.create inserting new tasks.

Before you start
  • PHP 8.5+ and Composer (Introduction — installation)
  • A terminal and text editor
  • New to PHP? Skim PHP basics first

What you will build#

Milestone You will…
1 Create deskflow-api with Composer
2 Ping the API and call the scaffold welcome.ping action
3 Add TaskService with a hardcoded task.list
4 Register task on MainSwitch
5 Create a tasks table and seed sample rows
6 Load tasks from SQLite in listAction
7 Add createAction and insert a row with curl

Later guides add validation rules, login, middleware, a React UI, and deploy — in the same repo you start here.

In DeskFlow…
DeskFlow — Northwind Studio’s internal task board. Services: task, member, project. Sample user: alex@northwind.studio. Dev server: http://127.0.0.1:8000.
flowchart LR A[composer create-project] --> B[php pionia serve] B --> C[TaskService] C --> D[SQLite tasks table] D --> E[task.list + task.create]

Checkpoint — what you built#

Action JSON body Result
Ping GET /api/v1/ping Health check
List tasks { "service":"task", "action":"list" } Rows from SQLite
Create task { "service":"task", "action":"create", "title":"…" } New row in tasks

Optional: run php pionia api:docs --ui and open http://127.0.0.1:8000/docs to see Moonlight catalog entries for your actions.

Common mistakes#

  • service not found — typo in MainSwitch alias, or PHP syntax error (check terminal where php pionia serve runs)
  • Connection refused — dev server not running, or wrong port (use 8000 from .env)
  • no such table: tasks — run php bin/init-db.php before calling task.list
  • Legacy uppercase keys — use lowercase "service" and "action" in JSON
  • Editing vendor/ — your code lives in services/, switches/, database/ only
